ExplanationUnderstanding the problemThe error 404 (Not Found) occurs because the browser is requesting a URL containing a double slash (//) in the path, which your server does not recognize. Looking at your request details:Path: //api/auth/telegramFull URL: https://...up.railway.app//api/auth/telegramThis usually happens when a base URL (ending with a /) is concatenated with a relative path (starting with a /). Most web servers treat // as a distinct path from /, leading to a 404 error because the route is not defined with that extra slash.How to verifyOpen the Network tab in Chrome DevTools.Locate the failed POST request.Check the General section for the "Request URL".Copy the URL and manually remove one of the slashes (e.g., change ...app//api... to ...app/api...) and test it in a tool like Postman or via curl. If it works with a single slash, the path concatenation in your code is the culprit.Suggested fixYou need to ensure that the base URL and the endpoint path do not both contain a slash at the joining point. You can either remove the trailing slash from your environment variable/configuration or sanitize the URL construction.If you are using axios or fetch with a base URL configuration:js// Fix: Ensure the base URL doesn't end with a slash if the endpoint starts with oneconst API_BASE_URL = "https://ara-ctw-telegrammini-card-production.up.railway.app"; // Removed trailing slash// Or use a helper to join them safelyconst endpoint = "/api/auth/telegram";const fullUrl = `${API_BASE_URL.replace(/\/$/, '')}/${endpoint.replace(/^\//, '')}`;// Example using axios// axios.post(fullUrl, data);Use code snippets with cautionSummaryProblem: The requested URL has an accidental double slash (//api/...), causing the server to return a 404.Cause: A trailing slash on the base URL merged with a leading slash on the API route.Fix: Remove the trailing slash from your base URL or use a logic check to prevent double slashes during URL concatenation.Data used to understand this message
